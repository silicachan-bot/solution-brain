以下是 B 站某视频评论区里的“评论对”。每一项都包含：
- 上文评论
- 针对它的回复评论

请只从“回复评论”里发现值得收录的语言模式。

优先保留这类模式：
- 适合在日常中文对话里当作回复复用
- 可以替换内容复用的句式模板
- 带有明确语气、立场或互动感
- 不是 AI 默认就很容易写出的普通表达
- 即使只出现一次，也能明显抽象成可迁移模板的回复句式

不要收录这类模式：
- 太普通的感叹、附和、吐槽
- 严重依赖视频内容、作品设定或评论区上下文才能成立的梗
- 更像围观评论，不像可迁移的对话回复
- 例句和模式本身对不上的内容
- 事实讨论、剧情分析、信息补充、长解释
- 对某个具体角色、剧情、事件做总结、安慰、解释后，硬抽出来的宽泛句子
- 把某人的成功、现象或结果归因到身份、家境、资源背景的“揭底句”
- 针对当前具体对象给建议、推荐替代去处、劝人降低预期的场景化回复
- 单纯的“哈哈 / 哈哈哈 / 谢谢 / 同问 / 没绷住 / 有点意思 / 安排”这类弱回复
- “回复 @某人 : …” 这种强依赖当前评论区链路的回复写法
- “火钳刘明 / 我能不能火就看你了 / 咱俩能不能火就看你了”这类更偏评论区互动、而不是对话回复的说法

宁缺毋滥。一批最多输出 3 个高质量模式；如果没有合适的，返回空数组 []。

对每个发现的模式，输出 JSON 数组，每个元素包含：
{
  "template": "含 [A] [B] 占位符的模板句",
  "examples": ["2-5个真实回复例句"],
  "description": "模式描述：是什么、什么时候能用于回复、传达什么感觉"
}

要求：
- `examples` 必须来自回复评论，不要引用上文评论
- 只有在回复里真的体现出稳定句式时才输出
- 如果只是单条评论有趣、但不像稳定可复用模式，不要输出
- 单例也可以输出，但前提是这句话本身有明显固定构式、梗感或可替换槽位
- 尽量把模式抽象成能复用的模板，而不是原句照抄
- 不要因为一句话“看起来能填 [A]”就硬抽模板；离开原评论话题就不自然的句子不要输出
- 优先保留“短、硬、像一句回话”的句子；谨慎保留宽泛安慰句、概括句、评价句
- 优先提取这种风格的模式：
  - `那我缺的[A]这一块谁给我补啊`
  - `大家好啊我是[A]，今天来点大家想看的[B]啊`
  - `[A]，这盛世如你所愿`
  - `哇还有[A]`
  - `[A]没这么[B]`
  - `啊呀，骇死我力！`
  - `这么强？！`
- 不要提取成这种风格：
  - `之前[A]的全还回去了`
  - `优秀的演员[A]`
  - `好歹[A]不是吗`
  - `[A]这一块，[B]这一块`
  - `首先你得有个[A]的身份`
  - `可以降低下预期，去[A]那`

只输出 JSON，不要其他文字。
