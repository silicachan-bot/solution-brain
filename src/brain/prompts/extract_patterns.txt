以下是 B 站某视频评论区里的"评论对"。每一项包含上文评论和针对它的回复评论。

请只从"回复评论"里提取值得收录的语言模式。把"是否收录"当作严格筛选题，而不是发散找亮点。

收录标准（需同时满足）：
- 像一句"回别人时会说的话"，有互动性，而不是围观、分析或补充信息
- 放到类似情绪、立场的其他对话里也成立，不依赖当前视频的专有内容——要独立评估回复本身，上文再奇特也不影响判断
- 有固定构式或明确槽位，且有口语惯用感或文化印记，不是任何母语者无需学习就自然会说出来的句子

排除标准：
- 只是普通中文语法结构（反问、假设、条件等）套成模板，没有固定说法或特殊语气
- 依赖当前视频内容、特定角色或梗，脱离原场景就不自然
- 事实讨论、信息补充、剧情分析、归因揭底、场景化建议

模板规则：
- 槽位替换后语感和特点必须保留；替换会让句子失去特色的，保留原句作为 template
- 保留原句骨架和口语感，不要改写成书面句
- 优先保留最小稳定单位，不要把核心短句包在多余的场景前缀里

宁缺毋滥。有多少真正符合标准的就输出多少。

对每个发现的模式，输出 JSON 数组，每个元素包含：
{
  "template": "含 [A] [B] 占位符的模板句，或无槽位的完整固定说法",
  "examples": ["2-5个逐字来自回复评论的原句"],
  "description": "详细描述这个用法：说话人在表达什么（情绪/立场/动作）；什么情境下会触发这句话；这个说法哪里有特点、为什么不是普通白话"
}

只输出 JSON，不要其他文字。
